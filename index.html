Infrastructure Documentation

Modpack Metadata Source

本ページはModpackのメタデータ管理構造を定義するものです。 ユーザー側の操作は必要なく、すべての構成管理は管理リポジトリ上のGitワークフローによって完結します。

Single Source of Truth

Source Control via Git

すべてのメタデータ、設定ファイル、およびModのリストはGitリポジトリで厳格に管理されています。 変更履歴はすべてコミットとして記録され、ロールバックや差分確認が容易です。

$ git commit -m "comment"
$ git push origin ****

Metadata Architecture

packwiz + TOML Format

管理者はpackwizを使用し、ModデータをTOML形式で保持します。 バイナリファイルではなくテキストベースで定義されるため、リポジトリの軽量化と可読性の向上を両立しています。

hash-format = "sha256"
[[files]]
file = "mods/example.pw.toml"
hash = "*******"
metafile = true

Management Principles

変更はすべてGitコミットによって追跡され、メタデータの整合性はpackwizによって検証されます。

TOMLファイルはModのメタデータ（名称、ID、ハッシュ値、URL）を正確に保持します。

配布用インデックス（index.toml）は、packwiz-installerが各クライアントを同期するために参照する唯一の定義です。

System: Metadata Repository · Protocol: Git/TOML · Engine: packwiz
